WDS vs EDS

Rev. 12/03/2024 LRM

Note: WDS and EDS can both be used for quantitative analysis. The methods are implemented in a few different ways on our microprobe, and users must choose between them -- or at least make decisions about when/whether to use these methods for quantification. WDS generally is more accurate and precise, but requires more time, training, and effort. EDS generally is faster and easier than WDS, but not quite as accurate or precise. As such, the choice of whether to use WDS or EDS can be fraimed as "is WDS worth the extra effort compared to EDS?"

Note: I make a few references below to larger numbers of analyses or transicts/grids working better for WDS than EDS. This is because the software for collecting EDS spectra (SEM Center) is intended for the manual placement of EDS analyses (i.e. clicking on the field of view with the mouse pointer) while the software for collecting WDS data (PC-EPMA) is intended for locating points within the context of the stage coordinate system. PC-EPMA actually CAN be used to collect EDS data, but I tend to trust the DTSA-II program more than JEOL's software because there is better documentation/community support for DTSA-II. More testing in this area is needed.

FAQs

Q: Can I get an analytical total with EDS?

A: Yes, EDS quantification can be done using standards, just like WDS. When a standards-based callibration is used, elemental concentrations are determined independently, so they can be summed to obtain an analytical total.


Q: Can I do standardless analyses using WDS?

A: Yes, the software on our microprobe includes a factory calibration for the WDS detectors, which is similar to that of the EDS detector. This is done by selecting "Cal-Std" in the "PC-EPMA" program.


Q: Can I use a combination of WDS and EDS in the same analysis?

A: Yes, this can be done using the "PC-EPMA" program.


Q: How do I do standards-based analyses using the EDS system?

A: Here are the options of which I am currently aware:

  • Use the "PC-EPMA" program as you would for WDS.
  • Collect spectra using the "SEM Center" program, and then post-process the data using the NIST "DTSA-II" program.
  • I think there is a way to save/use standards within the "SEM Center"/"SmileView" program(s). (???)

WDS summary

Advantages

  • Better accuracy
  • Lower detection limits
  • Simpler data reduction
  • Easier to explain
  • Better spectral resolution (less peak overlap)

Disadvantages

  • Calibration and validation take hours
  • More complicated software workflow
  • Many opportunities for user error
  • Poor temperature stability
  • Higher order X-ray artifacts

Recommended when...

  • required by the academic community
  • N > 100
  • Quantifying trace trace elements (< 1 wt%)
  • Elemental interferences are significant
  • User is a graduate student or faculty

Example use cases

  • Low-kV analyses (resolvable L-lines)
  • Thermobarometry involving trace elements (<1 wt%)
  • Monazite dating
  • Ti-in-Quartz analyses
  • Transects, grids, or other analyses for which the coordates need to be known or predetermined

EDS summary

Advantages

  • Fast (higher throughput and instant results)
  • Simpler/easier for qualitative analysis
  • Easy to train people
  • Elemental selection is not strictly required in advance

Disadvantages

  • Harder to explain
  • Relies on complicated, sometimes undocumented software
  • Poorer spectral resolution (more peak overlap)
  • Sum peak artifacts

Recommended when...

  • N < 100
  • Thermobarometry uses major elements (>10 wt%)
  • Standardization for LA-ICP-MS
  • Analyses are not likely to turn out well (missing elements, not smooth and flat)
  • User is an undergraduate student or engineer

Example use cases

  • Exploratory/preliminary analyses of unknown samples
  • “Routine analyses of major elements”
  • Mineral identification
