Drift correction and prevention

Rev. LRM 8/22/2024

Introduction

When you analyze the composition of a reference material on the microprobe, the difference between the reference composition and the measured composition is called “error.” Error is present in every analysis, and there are many sources of error – these can be categorized into two types: random error and systematic error. One type of systematic error is time-dependent “drift” which occurs when the state of the microprobe changes over time. Drift is mostly due to temperature changes, which cause the sample and spectrometer to change over time such that the alignment described by the Bragg equation is no longer present.

What can we do about drift?

Microprobe users can minimize errors due to drift by applying one or both of these two strategies: 1) drift prevention using analytical countermeasures, which change the way data are collected to minimize the impact of drift; or 2) drift correction by measuring replicates of a reference composition and correcting the results for drift.

Option 1: drift-prevention countermeasures

JEOL’s “PC-EPMA” program has two software functions that can be used to periodically readjust the analysis recipe to accommodate instrument drift: a) automatic peak searches and b) OM autofocusing.

Automatic peak searches minimize spectrometer drift by periodically realigning the spectrometer to the X-ray(s) used for the analyses. To enable automatic peak searches:

  1. Load the analysis recipe
  2. Check the “Detail” box for the purple “Quantitative analysis condition” section
  3. Change “Peak search of samples” to “Always” or “Every,” setting the frequency as appropriate. For example, if “Every” is selected and set to 4, peak searches will be performed on analysis numbers 1, 5, 9, 13…
  4. Go to the “Detail” view of the orange “Analysis element condition” section
  5. Set peak search from “0: Skip” to “1: Narrow” for each element as necessary. For example, Si is usually the element most affected by drift during analyses of Si minerals, and it is always present in the sample at high concentrations. As such, this is a good element for automatic peak searches while others might not be appropriate.
  6. Save the recipe, and start acquiring data.

OM autofocusing minimizes drift by automatically focusing the stage Z-height for individual samples before each analysis. To enable OM autofocusing:

  1. Load the analysis recipe
  2. Check the “Detail” box for the yellow “Analysis position condition” section
  3. For each analysis, check the box for “OM Auto Focus.” To enable this for many analyses at once, click the “Property” button on the right side of the yellow section and follow the prompts.
  4. Save the recipe, and start acquiring data.

Option 2: drift-correction

Alternatively, one can correct for drift by periodically analyzing replicates of a reference material and then calculating a time-dependent correction factor for one or several elements and then using this time-dependent correction factor to correct analyses of unknown materials. Additionally, we have an external data logger that can be used to monitor the room climate (temperature, pressure, relative humidity) to assist with these calculations.

Here’s the process I would recommend:

  1. Note the date and time.
  2. Start the green datalogger on the cart to the left of the column console:
    • Check that there is an SD card in the datalogger
    • Press and hold the “Log” button until you hear a beep and see "datalogger" appear on the screen.
  3. Prepare your quantitative analysis recipe and make sure to include analyses of a reference material (e.g. MAC Diopside) once every hour or so.
  4. Collect data as normal.
  5. When data collection is finished, stop the datalogger and retrieve the data:
    • press and hold the “Log” button on the datalogger to stop recording
    • Remove the SD card and copy the data onto your computer
    • Replace the SD card in the logger
  6. Talk to the lab manager about how to process the data, or you can do it yourself:
    • Make a plot of major elements in the reference materials vs temperature (or pressure, or humidity). Is there a correlation?
    • Re-plot the data for each major element as k = correct/measured, fit a line to this correlation. Now you can calculate a correction factor as a function of temperature (or pressure, or humidity).
    • Multiply this correction factor by your unknowns. Did this “flatten out” any drift-related variation? If so, pat yourself on the back, and go drink a beer!

Note: Correcting for drift in this way (i.e. by correcting elemental/oxide concentrations directly) is not completely rigorous because the corrected values will not have been re-corrected for matrix effects, and a more rigorous approach would be to drift correct the raw count rates (or K-ratios, as is done in the "Probe for EPMA" program) and then apply the matrix effect correction to the drift-corrected signal. While my proposed method of drift correction does incur this additional source of inaccuracy, it is unlikely to be significant compared to other sources of error -- at least in the case of a standard set of "major elements" analyzed for petrological purposes. This may not be the case for trace and light element analyses though.

Note: In theory, WDS quantitative analysis are superior to EDS analysis both in terms of precision and signl-to-noise ratios. However, a major drawback of WDS detectors compared to modern SDD-EDS detectors is their susceptability to drift due to changes in room climate. For example, rumor has it that even the heat generated by display monitors on the Control PC can influence count rates. In the past, I believed that drift was only significant on days when hot weather or afternoon thunderstorms overwhelmed the puny HVAC system at the NCFL. However, it is becoming more clear that drift can occur even on good days when the temperature stays between about 22 to 23 deg. C and temperature fluctions are within the 0.5 deg. C per hour facility specification stated by JEOL. For example, I've found that only about a 0.5 deg. C change in temperature translates into about a 1 wt% change in the measured SiO2 concentration in our MAC Diopside reference material. This means that on a typical day, we could easily have our room temperature stay within spec but have totals that go outside a desirable range of 99.0 to 101.0 wt%.
