WDS quantitative analyses (existing recipe)

Rev. 4/18/2024 LRM

Note: This SOP describes how to use a pre-calibrated “recipe” which has been calibrated in advance. For information about calibrating Quant analyses, see the “Quant setup” SOP.

  1. Click on the “Quick” tab in the PC-EPMA window (right side monitor), and double click on the User Recipe corresponding to the type of quantitative analysis you intend to do.
  2. Update the folder name and directory.
  3. In the yellow “sample locations” section for each sample location:

    Note: It’s a good practice to add to the list of points for each acquisition rather than deleting the locations of previously-analyzed points. You can simply uncheck points that you don’t want to analyze, and they will be omitted from the acquisition by the software.

    1. Select the next row in the list, and enter a comment to identify the sample
    2. Move to the correct location, and adjust the OM focus (stage Z-axis position)

      Note: always end on a clockwise turn and press the “Jog” button

    3. Click the “Read” button to retrieve the sample location from the current stage coordinates
    4. Set the correct beam conditions (e.g. a circular spot, 5 micron diameter)
    5. Click the “Apply” button, and double check that the information was stored correctly in the table.

  4. When you’re ready to start the analyses, note the calculated duration, Save the recipe, and click the “Acquire” button

To retrieve the data:

  1. Click the “EPMA Data” tab at the bottom of the PC-EPMA window
  2. Click the “Refresh” button to update the file list to include all new data
  3. Click the folder containing your data. You should see one or several numbered files which each correspond to a single analytical result.
  4. Click the “Summary” button near the folder icons.
  5. In the menu that appears, click the “To Excel” button. This will open the data in the Libre Calc spreadsheet program where data can be viewed and manipulated.

Note: Because the stage reproducibility in our system is imperfect, it is sometimes desirable to do analyses one at a time. This results in many project folders which each contain a single analysis file. To combine and export the data in this case, right click the parent directory and click the option to combine similar files. This will send all of the analyses into the same directory and renumber them so they can be easily exported as a single table.
